# Contributing

## Commit message rule

Use this format:

```text
${genre}: ${summary}
```

Then add one blank line, then write why this change is needed.

## Genre

Core genres:
- `feat` (feature)
- `bugfix`
- `docs`
- `refactor`
- `test`
- `chore`

You can add project-specific genres when they are clear and meaningful.

## Body

After the blank line, explain reason and why.
Keep it short and concrete.

Example body style:

nice
explanation
or
reason&why
here

## Examples

```text
feat: add project root picker
```

```text
bugfix: fix preview scroll bounds
```

```text
docs: document commit message rule
```

Full example:

```text
feat: add project root picker

nice
users can switch roots without restarting
reason&why
it reduces setup time for multi-repo work
```
